Cómo colaborar

El testeo de Freepto suele hacerse en todas las versiones beta y stable del proyecto.

Generalmente cada desarrollador prueba lo que hace, asís; que el equipo de testeo solo tiene que efectuar pruebas 'periódicas' para descubrir fallos eventuales.

En ocasiones los desarrolladores pueden pedir un test específico, por ejemplo, en el caso de modificaciones importantes del sistema o en cambios que pudieran depender del hardware usado. En este caso los desarrolladores crearán una nueva build y pidirán en la lista de correo general un comentario a los tester.

La coordinaciós;n se lleva a cabo a través de la lista de correo general. Si quieres unirte al equipo de testeo Â¡suscríbete a la lista y presentate!

Puedes encontrar las versiones de desarrollo y las versiones beta en esta página:

http://dev.freepto.mx/.

Luego puedes seguir la documentación para crear una memoria USB.

Tips&Tools

Para testear Freepto sin tener que realizar una instalación en un dispositivo, puedes usar qemu o VirtualBox para emular el sistema. De esta manera no será posible testear el funcionamento de la persistencia cifrada.

qemu/kvm

Ejemplo para sistemas x86_64:

qemu-system-x86_64 -cpu kvm64 -enable-kvm freepto-es_ES-0.1.2.img

VirtualBox

Para crear una maquina virtual con virtualbox puedes descargar el script vboxtesting.

Ejemplo para ejecutarlo desde CLI:

wget http://raw.github.com/AvANa-BBS/freepto-lb/master/tools/vboxtesting.sh

chmod +x vboxsetting.sh

bash vboxtesting.sh
